\doxysubsection{Detailed Description}
Fingerprint sensor driver (ZW111) for ESP32. 

\hypertarget{fingerprint_8h_autotoc_md0}{}\doxysubsubsection{\texorpdfstring{1. Introduction}{1. Introduction}}\label{fingerprint_8h_autotoc_md0}
This driver provides an interface for the ZW111 fingerprint sensor, enabling fingerprint enrollment, matching, and template storage. It is designed for embedded systems using the ESP32 microcontroller, providing an event-\/driven architecture for efficient processing.\hypertarget{fingerprint_8h_autotoc_md1}{}\doxysubsubsubsection{\texorpdfstring{1.\+1 Supported Platforms}{1.\+1 Supported Platforms}}\label{fingerprint_8h_autotoc_md1}

\begin{DoxyItemize}
\item ESP32-\/based development boards
\item ESP-\/\+IDF (Espressif IoT Development Framework)
\end{DoxyItemize}\hypertarget{fingerprint_8h_autotoc_md2}{}\doxysubsubsubsection{\texorpdfstring{1.\+2 Compiler and Toolchain}{1.\+2 Compiler and Toolchain}}\label{fingerprint_8h_autotoc_md2}

\begin{DoxyItemize}
\item GCC (ESP32 toolchain)
\item Compatible with ESP-\/\+IDF v4.\+x or later
\end{DoxyItemize}\hypertarget{fingerprint_8h_autotoc_md3}{}\doxysubsubsubsection{\texorpdfstring{1.\+3 Dependencies}{1.\+3 Dependencies}}\label{fingerprint_8h_autotoc_md3}

This system operates on an event-\/driven architecture where each key operation, such as image capture, feature extraction, or match status, triggers specific events. The events help the application know when an operation is complete or has failed. The following events are generated based on the corresponding system actions.\hypertarget{fingerprint_8h_autotoc_md10}{}\doxysubsubsubsection{\texorpdfstring{3.\+1 Event Handler}{3.\+1 Event Handler}}\label{fingerprint_8h_autotoc_md10}
Event handler for processing fingerprint events.

This function processes various fingerprint-\/related events and logs messages based on the event type. It can be used as a sample for handling fingerprint events in the application. The events are processed based on the {\ttfamily \doxylink{fingerprint_8h_a6a07f45ee6da95e2ccbcbb0e0699f5ae}{fingerprint\+\_\+event\+\_\+type\+\_\+t}} enumeration, and corresponding messages are logged using the ESP-\/\+IDF logging system.


\begin{DoxyParams}{Parameters}
{\em event} & The fingerprint event that occurred. This can be one of the following\+:
\begin{DoxyItemize}
\item {\ttfamily EVENT\+\_\+\+FINGER\+\_\+\+DETECTED}\+: Triggered when a finger is detected.
\item {\ttfamily EVENT\+\_\+\+IMAGE\+\_\+\+CAPTURED}\+: Triggered when the fingerprint image is captured successfully.
\item {\ttfamily EVENT\+\_\+\+FEATURE\+\_\+\+EXTRACTED}\+: Triggered when the fingerprint features are extracted successfully.
\item {\ttfamily EVENT\+\_\+\+MATCH\+\_\+\+SUCCESS}\+: Triggered when a fingerprint match is successful.
\item {\ttfamily EVENT\+\_\+\+MATCH\+\_\+\+FAIL}\+: Triggered when a fingerprint mismatch occurs.
\item {\ttfamily EVENT\+\_\+\+ERROR}\+: Triggered for general errors during fingerprint processing.
\end{DoxyItemize}\\
\hline
\end{DoxyParams}
